[HBase] 才发现HBase REST服务占用的是8080端口

时间： 2016-03-03   |   分类： Hadoop/Spark   HBase     |   阅读： 70 字 ~1分钟

今天用一下Tomcat，结果发现8080端口还被占了，谁呀？ 竟然是HBase REST服务占用的！！看了一下Ports Used by Components of CDH 5，发现ClouderaCDH里是这么用的： - 8080：Non- Cloudera Manager - managed HBase REST Service - 20550：Cloudera Manager - managed HBase REST Service - 8085：HBase REST UI

8080端口还是留着吧，对hbase-site.xml做如下修改： 重启HBase REST服务：

hbase-daemon.sh stop rest
hbase-daemon.sh start rest

通过HBase REST UI检查，REST服务端口改成了20550： 另一种修改REST服务端口的方式是在启动HBase REST服务命令时通过-p选项直接指定端口。例如：

hbase-daemon.sh start rest -p 20550
